### Вмикання юніт- і компонент-тестів

Для того щоб зробити юніт тестування доступним (для побудови і запуску) необхідно переконфігурувати проект з увімкненою опцією `ENABLE_UNIT_TESTS` за допомогою команди (для GNU/Linux):

```
# у кореневій директорії проекту

cmake -B build -S . -DENABLE_UNIT_TESTS=ON && cmake --build build
```

Після чого можна скористатися утилітою `ctest`.

Компонентні тести вмикаються аналогічно через опцію `ENABLE_COMPONENT_TESTS` і можуть бути поєднані з юніт-тестами:

```
# у кореневій директорії проекту

cmake -B build -S . -DENABLE_UNIT_TESTS=ON -DENABLE_COMPONENT_TESTS=ON && cmake --build build
```

Компонентні тести віджетів GTKmm потребують наявного сервера дисплея для запуску; у безекранному (headless) середовищі вони пропускаються автоматично.
